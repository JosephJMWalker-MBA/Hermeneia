## Newly Proven

### CI-014 — Monotonic Supersession

Hermeneia can now supersede an object by appending a relation only.

The original object remains queryable. Multiple superseding descendants may
coexist. Historical chains remain traversable. The supersession relation itself
is append-only: `UPDATE` and `DELETE` are rejected by database triggers.

### CI-015 — Anti-Helpfulness Compliance

Hermeneia now has executable tests proving the implementation does not improve
evidence before persistence.

Verified facts:

- `SourceExtraction.raw_text` equals parser output exactly.
- pathological parser text survives SourceExtraction persistence unchanged;
- pathological PDF parser output is the storage oracle;
- `Observation` stores raw selected text, not normalized text;
- normalization is isolated to `ObservationDerived`;
- sentence splitting no longer strips returned Observation text;
- downstream layers cannot write evidence tables;
- parser and SourceExtraction compiler contain no text-repair helpers.
