# Hermeneia Experiment Matrix

**Status:** Active research program  
**Updated:** 2026-06-25

---

## Purpose

Each experiment runs the full Hermeneia pipeline on a corpus and records:

- Which observations were selected
- Which questions were posed
- How evidence was weighted
- What interpretations were constructed
- How the expression compressed understanding

Experiments are compared across two axes: **language** and **domain**.

Language variation tests whether the architecture preserves understanding across expressive forms.

Domain variation tests whether the cognitive architecture is general — the same pattern of responsibilities appearing in literary, theological, legal, scientific, and organizational inquiry.

## Replicated Findings (Language Axis)

The following findings are supported by all three completed experiments and satisfy the threshold for **Replicated Finding** status:

| Finding | Description | Evidence |
|---|---|---|
| **RF-001** | Observation stability | Same major textual phenomena identified across English, Spanish, and Mandarin investigations |
| **RF-002** | Classification stability | Same literary device categories applied to the same phenomena across all three |
| **RF-003** | Question divergence | Governing questions consistently differ across cultural contexts; question selection is the primary culture-adaptive variable |

To graduate these findings to **Established Pattern**, they must also hold across the domain axis (3+ distinct domains).

## Measurement Protocol

For each experiment, record:

1. **Observation stability** — which textual phenomena were identified; compare to prior experiments on the same corpus
2. **Classification stability** — how phenomena were categorized; compare to prior experiments
3. **Question selection** — the governing investigative question; compare across language/domain variants
4. **Evidence weighting** — which observations received higher salience; represent as priority vectors where possible
5. **Evidence network structure** — causal connections between observations; compare graph topology
6. **Compression** — the final summary sentence or thesis
7. **Compression asymmetry** — whether the execution remained agent-centered or generalized to relationship/community

---

## Evidential Standards

Findings from this matrix will be classified as:

- **Empirical Finding** — observed in one or more experiments; not yet replicated sufficiently for general conclusion
- **Replicated Finding** — observed consistently across 3+ experiments in the same axis
- **Established Pattern** — observed across multiple axes (language AND domain); sufficient for inclusion in research claims

No finding graduates to the white paper without at least Empirical Finding status. No finding is called a general principle without Replicated or Established Pattern status.

## The Research Questions This Matrix Addresses

**Primary:**
1. How does understanding evolve?
2. What conditions lead different investigators to evolve understanding differently from the same evidence?

**Secondary:**
3. Is the cognitive architecture (discovery → reconstruction → communication → verification → governance) scale-independent?
4. Is question selection the primary culture-adaptive variable, with evidence weighting downstream?
5. Can investigative emphasis be represented as a measurable quantitative profile?

---

## Connection to Architecture

Each experiment is simultaneously a research investigation and an architectural stress test.

If the pipeline decomposition holds across domains, the architecture has generalized beyond its origin.

If it does not, the failure is a specification — it tells the next architecture version exactly what to account for.

This is what Hermeneia means by making inquiry inspectable: the research program applies to itself.
